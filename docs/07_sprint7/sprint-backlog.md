# スプリント7 バックログ(認証・手作り)

## スプリントゴール

**「アカウントの登録・ログイン・ログアウトができ、未ログインではTODO機能に
アクセスできない(ログイン画面へ誘導される)状態にする」**

対応するユーザーストーリー: [US-11/US-12](../00_project/user-stories.md) / PBI #20
写経目安時間: **3時間**

## このスプリントの位置づけ

認証(誰であるか)に集中する。**認可(誰のTODOか)はスプリント8**。
S7完了時点では「ログインは必要だが、全ログインユーザーが全TODOを見る」中間状態。
この分割は、認証と認可という別概念を混ぜずに学ぶための意図的なもの。

Breezeは使わず**手作り**(Tailwind/Alpine依存を避け、仕組みを露出させる)。

## 前スプリントのTryをこのスプリントで使う

- **T-13**: 大改修の前に「どの既存テストが赤くなるか」を予想し、予想と実際を突き合わせる
- **T-14**: ファイル削除時は `git status` で影響を確認
- **T-15**: テストデータの文字列は部分一致の罠を避ける

## スコープ外

- パスワードリセット(メール送信が絡む。教材の焦点外)
- メール確認(email_verified)
- 「ログイン状態を保持」(remember me)
- TODOの所有権・他人のTODO制限(スプリント8)

## リスクと対策

- **既存24テストが軒並み赤くなる** → 想定内。T-13で予想してから、
  `actingAs($user)` でログイン状態を作る形に更新する。テストは仕様の写し鏡
- パスワードを平文保存してしまう事故 → `Hash::make` を使い、実験7-Aで
  ハッシュ保存を実データ確認する
