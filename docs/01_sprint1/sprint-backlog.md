# スプリント1 バックログ

## スプリントゴール

**「リポジトリをクローンした写経者が、READMEの手順だけで `http://localhost:8080` に
Laravelの初期画面を表示できる状態にする」**

対応するユーザーストーリー: [US-0(開発環境の再現)](../00_project/user-stories.md)、
[US-7(教材)](../00_project/user-stories.md) の一部

対応するPBI: #1(Sail環境構築)、#2(README)、#3(GitHub運用開始)

写経目安時間: **1.5〜2時間**

## タスク一覧

| # | タスク | 完了条件 |
|---|---|---|
| T1 | スプリントバックログ作成(このファイル) | ゴール・タスク・完了条件が明文化され、mainにコミットされている |
| T2 | 作業ブランチ `feature/sprint1-environment` を作成 | ブランチ上で作業が始められる |
| T3 | Composer入りコンテナで Laravel 本体を導入し、`sail:install` で `compose.yaml`(MySQL構成)を生成 | `vendor/bin/sail` が存在し、`compose.yaml` にmysqlサービスがある |
| T4 | `.env` / `.env.example` をプロジェクト用に整備(APP_PORT=8080 / VITE_PORT=5180 / COMPOSE_PROJECT_NAME / アプリ名・URL・DB名) | `.env.example` をコピーするだけで本プロジェクト用の設定になる |
| T5 | `sail up -d` で起動し、Welcome画面とDBマイグレーションを確認 | ブラウザで `http://localhost:8080` が200で表示され、`migrate` が成功する |
| T6 | わざと失敗実験: APP_KEY を空にして500エラーを観察し、ログから原因を辿る | エラーの読み方(1行目から読む)がトレースガイドに記録されている |
| T7 | README にクローン直後からのセットアップ手順を記載(鶏と卵問題の解決手順含む) | 手順だけで環境構築が再現できる(vendorなし状態からの手順がある) |
| T8 | 開発トレースガイド・概念解説集のスプリント1分を執筆 | 各コミットのステップが必須要素(足場の表・動作確認・ここでコミット等)を満たす |
| T9 | PR作成 → マージ → **マージ後のmainで動作確認** | PRがマージされ、main上で `sail up -d` → Welcome画面表示を再確認済み |
| T10 | スプリントレビュー記録・レトロスペクティブ記録を作成 | トラブルと調査過程が記録され、Try が次スプリントで使える形になっている |

## リスクと対策

- **ポート衝突**: 事前調査で `VITE_PORT=5174` が使用中と判明 → `5180` に変更済み
  (→ [qa-log.md](../00_project/qa-log.md) に記録)
- **Dockerイメージのダウンロード時間**: 初回は数百MB〜のダウンロードが発生する。
  写経目安時間には通信待ちを含む
