# スプリント8 バックログ(認可+負債返済+教材最終化)

## スプリントゴール

**「TODOに所有者(user_id)を持たせ、自分のTODOだけを扱える(他人のは403)ようにし、
バリデーション重複をFormRequestで回収して、第2部の教材を完成させる」**

対応するユーザーストーリー: [US-13](../00_project/user-stories.md) / PBI #21・#22
写経目安時間: **3時間**

## このスプリントの3本柱

1. **認可(Policy)**: 認証(S7)に続き「誰が何を触れるか」。他人のTODOは403
2. **技術的負債の返済**: 第1部で意図的に残したバリデーション重複をFormRequest化
3. **教材最終化v2**: 第2部の対応マップ・トラブル一覧・READMEを完成させる

## 前スプリントのTryをこのスプリントで使う

- **T-16**: user_id追加は「カラム追加→既存行をデモに紐付け→NOT NULL化」を1マイグレーションで
- **T-17**: 認可テストは「本人はできる/他人は403」を必ずペアで書く
- **T-18**: FormRequest化は「緑→変更→緑」で振る舞いを変えないことを担保する

## スコープ外

- 共有・チーム機能(他人にTODOを見せる)
- 管理者ロール(全TODO閲覧)
- ソフトデリート

## リスクと対策

- **user_id NOT NULL化で既存行が制約違反** → T-16どおり、移行(デモ紐付け)を
  制約付与の前に同一マイグレーションで行う
- **認可漏れ(Policyの当て忘れ)** → 各操作にauthorizeを付け、実験8-Aで
  「Policyを外すと他人のが見える」ことを観察して効き目を確認
- FormRequest化で振る舞いが変わる → T-18(緑→変更→緑)で担保
